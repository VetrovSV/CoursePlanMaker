# Описание показателей (дескрипторов) и критериев оценивания компетенций на различных этапах их формирования

Контроль качества освоения дисциплины включает в себя текущий контроль
успеваемости и промежуточную аттестацию. Текущий контроль успеваемости
и промежуточная аттестация обучающихся проводятся в целях установления
соответствия достижений обучающихся поэтапным требованиям
образовательной программы к результатам обучения и формирования
компетенций.

| Модуль | Контролируемые разделы (темы) дисциплины | Код контролируемой компетенции (или её части) | Наименование оценочного средства |
|---|---|---|---|

### Критерии и шкала оценивания лабораторных работ

| Оценка | Критерий оценки |
|---|---|
| «Зачтено» | — Работа полностью выполнена в соответствии с заданием. |
| | — Даны ответы на теоретические вопросы по теме работы и вопросы о содержании лабораторной работы (SQL-запросов, схемы БД). Аргументирован выбор структур данных, типов и ограничений. |

## Критерии и шкалы оценивания результатов обучения при проведении промежуточной аттестации

Промежуточная аттестация предназначена для определения уровня освоения
всего объема учебной дисциплины. Форма промежуточной аттестации —
экзамен.

Для оценивания результатов обучения при проведении промежуточной
аттестации используется четырёхбалльная шкала: «Отлично», «Хорошо»,
«Удовлетворительно», «Неудовлетворительно».

| Шкала оценивания | Критерии оценивания | Уровень освоения компетенций |
|---|---|---|
| **Отлично** | Обучающийся правильно ответил на теоретические вопросы. Показал отличные знания в рамках учебного материала, свободно ориентируется в моделях данных, языке SQL, принципах проектирования и нормализации. Правильно выполнил практические задания. Ответил на все дополнительные вопросы. | Эталонный |
| **Хорошо** | Обучающийся с небольшими неточностями ответил на теоретические вопросы. Показал хорошие знания в рамках учебного материала. С небольшими неточностями выполнил практические задания. Ответил на большинство дополнительных вопросов. | Стандартный |
| **Удовлетворительно** | Обучающийся с существенными неточностями ответил на теоретические вопросы. Показал удовлетворительные знания в рамках учебного материала. С существенными неточностями выполнил практические задания. Допустил много неточностей при ответе на дополнительные вопросы. | Пороговый |
| **Неудовлетворительно** | Обучающийся при ответе на теоретические вопросы и при выполнении практических заданий продемонстрировал недостаточный уровень знаний и умений. При ответах на дополнительные вопросы было допущено множество неправильных ответов. | Компетенции не сформированы |

# Типовые контрольные задания или иные материалы, необходимые для оценки знаний, умений, навыков и (или) опыта деятельности, характеризующих этапы формирования компетенций в процессе освоения образовательной программы

## Оценочные средства текущего контроля успеваемости

### Лабораторная работа №1. Установка и настройка СУБД

**Задание:**
1. Установите СУБД PostgreSQL на рабочую станцию.
2. С помощью DBeaver создайте новую базу данных с именем `lab1_student`
   (где `student` — ваша фамилия латиницей).
3. Создайте таблицу `Студенты` со следующими полями:
   - `id` — первичный ключ (SERIAL),
   - `фамилия` — VARCHAR(50),
   - `имя` — VARCHAR(50),
   - `отчество` — VARCHAR(50),
   - `дата_рождения` — DATE,
   - `номер_группы` — VARCHAR(10).
4. Заполните таблицу 5–7 записями.
5. Выполните запросы:
   - выбрать всех студентов заданной группы;
   - выбрать студентов, родившихся после 01.01.2000;
   - обновить номер группы для одного из студентов;
   - удалить одну запись.

**Контролируемые компетенции:** ОПК-1, ОПК-8.

### Лабораторная работа №2. Проектирование и создание БД

**Задание:**
1. Постройте ER-диаграмму для предметной области «Учебный процесс»,
   которая включает сущности:
   - `Преподаватели` (id, фамилия, имя, отчество, кафедра, должность),
   - `Дисциплины` (id, название, код, часы, семестр),
   - `Группы` (id, название, курс, факультет),
   - `Студенты` (id, фамилия, имя, отчество, группа),
   - `Успеваемость` (id, студент, дисциплина, преподаватель, оценка, дата).
2. Определите связи между сущностями (1:1, 1:M, M:M).
3. Приведите схему как минимум к 3-й нормальной форме.
4. Реализуйте схему БД в PostgreSQL с учётом:
   - первичных и внешних ключей,
   - ограничений NOT NULL, UNIQUE, CHECK,
   - индексов для часто запрашиваемых полей.
5. Заполните таблицы тестовыми данными (не менее 5 записей в каждой).

**Контролируемые компетенции:** ОПК-1, ПК-1.

### Лабораторная работа №3. SQL-запросы

**Задание:**
Используя схему БД из лабораторной работы №2, выполните следующие
запросы:

1. Вывести список студентов указанной группы с сортировкой по фамилии.
2. Для каждого студента вывести среднюю оценку по всем дисциплинам.
3. Вывести преподавателей, у которых средняя оценка студентов выше 4.0.
4. Вывести дисциплины, которые читаются на указанном курсе, и
   количество студентов, изучающих их.
5. Используя JOIN, вывести полную информацию об успеваемости: ФИО
   студента, название дисциплины, ФИО преподавателя, оценку.
6. Используя подзапрос, найти студентов, чья оценка по дисциплине выше
   средней оценки по этой дисциплине.
7. Используя оконные функции, для каждого студента вывести его оценку
   по дисциплине, а также среднюю оценку по группе и максимальную
   оценку по дисциплине.

**Контролируемые компетенции:** ОПК-8.

### Лабораторная работа №4. Хранимые процедуры и триггеры

**Задание:**
1. Создайте хранимую процедуру `добавить_оценку`, которая принимает
   id студента, id дисциплины, id преподавателя и оценку, проверяет
   корректность данных и добавляет запись в таблицу `Успеваемость`.
2. Создайте функцию `средняя_оценка_студента`, которая принимает id
   студента и возвращает его среднюю оценку.
3. Создайте триггер, который при добавлении новой оценки проверяет,
   что оценка находится в диапазоне от 2 до 5. В случае некорректного
   значения — отменяет операцию и возвращает сообщение об ошибке.
4. Продемонстрируйте работу транзакций:
   - начните транзакцию;
   - выполните вставку нескольких записей;
   - создайте точку сохранения (SAVEPOINT);
   - выполните откат к точке сохранения;
   - завершите транзакцию.

**Контролируемые компетенции:** ОПК-1, ОПК-8, ПК-1.

## Оценочные средства промежуточной аттестации

### Экзаменационные вопросы

1. Понятие базы данных и системы управления базами данных. Архитектура
   СУБД.
2. Модели данных: реляционная, иерархическая, сетевая, NoSQL.
   Сравнительная характеристика.
3. Реляционная модель данных: основные понятия (отношение, кортеж,
   атрибут, домен, ключ).
4. Этапы проектирования баз данных. Концептуальное, логическое и
   физическое проектирование.
5. ER-моделирование: сущности, атрибуты, связи. Типы связей (1:1, 1:M,
   M:M).
6. Нормализация таблиц: 1НФ, 2НФ, 3НФ, НФБК. Примеры приведения к
   нормальным формам.
7. Денормализация: цели и последствия.
8. Язык SQL. Стандарты SQL. Классификация операторов (DDL, DML, DCL,
   TCL).
9. DDL: CREATE, ALTER, DROP. Создание и модификация таблиц, ограничения
   целостности.
10. DML: SELECT, INSERT, UPDATE, DELETE. Синтаксис и примеры.
11. Оператор SELECT: WHERE, ORDER BY, GROUP BY, HAVING.
12. Агрегатные функции: COUNT, SUM, AVG, MIN, MAX.
13. JOIN-операции: INNER JOIN, LEFT JOIN, RIGHT JOIN, FULL OUTER JOIN,
    CROSS JOIN.
14. Вложенные запросы (подзапросы): коррелированные и некоррелированные.
15. Оконные функции: OVER, PARTITION BY, ROW_NUMBER, RANK, DENSE_RANK.
16. Представления (VIEW): создание, использование, преимущества.
17. Индексы: назначение, типы (B-tree, hash, GiST, GIN). Влияние на
    производительность.
18. Транзакции: понятие, свойства ACID. BEGIN, COMMIT, ROLLBACK,
    SAVEPOINT.
19. Уровни изоляции транзакций: READ UNCOMMITTED, READ COMMITTED,
    REPEATABLE READ, SERIALIZABLE.
20. Блокировки: разделяемые и исключительные. Блокировки на уровне
    строк и таблиц.
21. Хранимые процедуры и функции: создание, параметры, возвращаемые
    значения.
22. Триггеры: назначение, типы (BEFORE, AFTER, INSTEAD OF). Примеры
    использования.
23. Резервное копирование и восстановление баз данных. Логическое и
    физическое резервирование.
24. Методы обеспечения безопасности данных: аутентификация,
    авторизация, аудит.
25. NoSQL-базы данных: типы (документные, графовые, key-value,
    колоночные), области применения.
26. Особенности проектирования баз данных для промышленных
    автоматизированных систем.
27. Интеграция баз данных с прикладными модулями АСУТП и
    SCADA-системами.
28. Методы оптимизации производительности запросов: анализ плана
    выполнения, использование индексов.

# Описание процедур проведения промежуточной аттестации

## Экзамен

При определении уровня достижений обучающихся на экзамене учитывается:

- знание программного материала и структуры дисциплины;
- знания, необходимые для решения типовых задач, умение выполнять
  предусмотренные программой задания;
- владение методологией дисциплины, умение применять теоретические
  знания при решении задач, обосновывать свои действия.

Проведение промежуточной аттестации в форме экзамена позволяет оценить
уровень сформированности компетенций по результатам текущего контроля
и ответа на экзамене.

Экзамен проводится в форме собеседования по экзаменационным билетам.
Каждый билет содержит два теоретических вопроса из перечня и одно
практическое задание (написание SQL-запроса или проектирование схемы
БД). Перечень теоретических вопросов и типов практических заданий
обучающиеся получают в начале семестра.

| Условие получения оценки | Оценка |
|---|---|
| Успешная защита всех лабораторных работ, отличный ответ на экзаменационные вопросы, выполнение практического задания без ошибок | Отлично |
| Успешная защита всех лабораторных работ, хороший ответ на экзаменационные вопросы, выполнение практического задания с незначительными неточностями | Хорошо |
| Успешная защита не менее 3 лабораторных работ, удовлетворительный ответ на экзаменационные вопросы, выполнение практического задания с существенными неточностями | Удовлетворительно |
| Невыполнение лабораторных работ или неудовлетворительный ответ на экзаменационные вопросы, невыполнение практического задания | Неудовлетворительно |
