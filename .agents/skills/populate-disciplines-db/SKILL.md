# Наполнение БД дисциплинами из учебного плана

## Когда использовать

После того, как в БД добавлен учебный план (запись в `study_plans`), нужно наполнить
его дисциплинами из соответствующего PDF-файла учебного плана.

## 1. Извлечь данные из PDF

```sh
pdftotext -layout исходные_данные/учебные_планы/<файл>.pdf -
```

Просмотреть вывод и найти таблицу дисциплин. В PDF таблица может быть растянута
по ширине страницы. Ищи колонки:

| Колонка | Что ищем |
|---|---|
| Индекс | Шифр дисциплины (Б1.О.01.01, Б1.В.ДВ.01.01 и т.п.) |
| Наименование | Название дисциплины (может быть на нескольких строках) |
| Экзамены/Зачёты | Номер семестра под соответствующей колонкой |
| Часы по семестрам | В правой части таблицы — колонки `лк`, `лб`, `пр`, `ср` по каждому семестру |
| ЗЕТ | В колонке «Трудоёмкость в ЗЕТ» (после часов) |
| Кафедра | Последняя колонка |

Сложности:

- Названия дисциплин могут **разрываться переносом строки** — собрать вручную.
- Часы по семестрам могут быть **сдвинуты** из-за форматирования — сверять визуально с PDF.
- Для модулей (Б1.О.01, Б1.В.01) итоговые строки — не добавлять как отдельные дисциплины,
  только вложенные конкретные дисциплины.

## 2. Создать скрипт наполнения

Создай файл `db/populate_<описание>.py` по образцу `db/populate_magistracy_2025.py`.

Заготовка:

```python
import os, sys
sys.path.insert(0, os.path.dirname(os.path.abspath(__file__)))
from create_db import (
    create_db, add_discipline, add_discipline_to_plan, find_plan_id,
)

conn = create_db()
plan_id = find_plan_id(conn, <год>, "<форма>", "<код направления>", "<профиль>")
# или получить plan_id иначе

disciplines_data = [
    # (код, название, семестр, всего_ч, лк, лаб, пр, срс, экз_ч, зет, форма_аттестации)
    ("Б1.О.01.01", "Пример", 1, 144, 16, 16, 0, 112, 0, 4, "экзамен"),
]

for code, name, sem, h_total, h_lk, h_lb, h_pr, h_srs, h_exam, credits, exam_form in disciplines_data:
    disc_id = add_discipline(conn, code, name)
    add_discipline_to_plan(conn, plan_id, disc_id, sem,
        hours_total=h_total, hours_lecture=h_lk, hours_lab=h_lb,
        hours_practice=h_pr, hours_self_study=h_srs,
        hours_exam=h_exam, credits=credits, exam_form=exam_form)

conn.commit()
conn.close()
```

### Допустимые значения `exam_form`

- `"зачёт"`
- `"экзамен"`
- `"зачёт с оценкой"`
- `None` — для ГИА (ВКР)

## 3. Запустить скрипт

```sh
python3 db/populate_<описание>.py
```

## 4. Проверить результат

```sh
sqlite3 db/disciplines.db "
SELECT dip.semester, d.code, d.name, dip.hours_total, dip.credits, dip.exam_form
FROM disciplines_in_plans dip
JOIN disciplines d ON d.id = dip.discipline_id
WHERE dip.plan_id = <ID_плана>
ORDER BY dip.semester, d.code;
"
```

### Что проверять

- Все ли дисциплины из PDF попали в вывод.
- Совпадают ли семестры.
- Совпадают ли ЗЕТ.
- **Сходятся ли суммы**: часы неотрицательные, exam_form указана у всех, кроме ВКР.

## 5. Если дисциплина идёт несколько семестров (практики)

Практики (Б2) и НИР могут быть распределены на несколько семестров.
Добавляй отдельную запись для каждого семестра.
Основные часы/ЗЕТ — в первый семестр, в остальные — 0 часов и 0 ЗЕТ.

### Пример

```python
extra_data = [
    ("Б2.О.01(П)", "Производственная практика (технологическая)", 3, 0, 0, 0, 0, 0, 0, 0, None),
]
```

## Референс

- **База данных**: `db/disciplines.db`
- **Хелперы**: `db/create_db.py` (функции `add_discipline`, `add_discipline_to_plan`, `find_plan_id`)
- **Пример**: `db/populate_magistracy_2025.py`
- **PDF учебных планов**: `исходные_данные/учебные_планы/`
