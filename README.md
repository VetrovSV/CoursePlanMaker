# CoursePlanMaker

**Генератор рабочих программ дисциплин (РП) и фондов оценочных средств (ФОС) для вузов.**

Проект создан, чтобы автоматизировать рутинную работу преподавателя — составление документов по учебным дисциплинам. Всё, что раньше делалось в Word вручную, теперь генерируется через диалог с ИИ-ассистентом.

---

## Зачем это нужно?

Преподаватель вуза каждый год обновляет десятки документов:

- **Рабочая программа дисциплины (РП)** — что изучаем, сколько часов, какие темы, как оцениваем
- **Фонд оценочных средств (ФОС)** — конкретные задания, тесты, вопросы к зачёту/экзамену

Раньше: открыть Word → скопировать прошлогодний → поменять даты → пересчитать часы вручную → забыть что-то → переделывать.

Теперь: сказать агенту «Создай РП для дисциплины Х» → ответить на несколько вопросов → получить готовый файл.

---

## Что такое «агент» и как он работает?

**Представьте, что у вас есть стажёр, который:**

- Умеет читать шаблоны документов — и никогда не забывает структуру
- Может искать информацию в интернете (ФГОСы, списки литературы, профстандарты)
- Знает, что часы в таблицах должны сходиться и проверяет это сам
- Не выдумывает данные, а честно спрашивает, если не знает
- Работает внутри редактора Zed, прямо рядом с вашим кодом

Вот как это выглядит:

```
Вы:     «Создай РП для дисциплины "Теория вычислительных процессов"»
Агент:  «Какие часы? Какой семестр? Какие компетенции?»
Вы:     «144 часа, 6 семестр, ОПК-1, ОПК-2»
Агент:  (создаёт файл, проверяет суммы, показывает результат)
```

Агент использует **навыки (skills)** — записанные сценарии рабочих процессов. Когда вы просите создать РП, агент загружает пошаговую инструкцию и действует строго по ней.

## Пайплайн: как выглядит создание документа

```
┌────────────┐      ┌──────────────────────┐     ┌──────────────┐
│ Вы         │      │                      │     │              │
│ говорите:  │─────►│      Zed Agent       │─────►│ Готовый файл │
│ «Создай    │      │                      │     │              │
│  РП»       │      │   с навыком          │     │ РП.{Дис}.{Год}.md
└────────────┘      │   create-rp          │     └──────────────┘
                    │                      │
                    └──────────┬───────────┘
                               │
                    ┌──────────▼───────────┐     ┌──────────────┐
                    │                      │     │              │
                    │    Web search        │─────►│ Компетенции  │
                    │    (по запросу)      │     │ Литература   │
                    │                      │     │ Стандарты    │
                    └──────────────────────┘     └──────────────┘
```

**Пошагово:**

1. **Вы говорите** агенту, какую дисциплину нужно создать (название, часы, семестр, направление)
2. **Агент задаёт вопросы** — если данных не хватает (цели, задачи, ФИО, литература, компетенции)
3. **Ищет в интернете** — ФГОС-компетенции, учебники на urait.ru, профстандарты
4. **Создаёт файл РП** — заполняет шаблон, таблицы, проверяет суммы часов
5. **Валидирует** — запускает скрипт проверки (часы сходятся? все поля заполнены?)
6. **Вы получаете** готовый Markdown-файл, который можно открыть, поправить и отправить

Если нужно ещё и **ФОС** — агент создаёт его на основе уже готовой РП: берёт те же компетенции, темы, часы и формирует оценочные средства.

## Что нужно сделать, чтобы начать работу

### 1. Установить Zed

Скачайте редактор [Zed](https://zed.dev). Это редактор кода, похожий на VS Code, но быстрее и со встроенной поддержкой ИИ-агентов.

### 2. Настроить подключение к ИИ (LLM)

Агент использует нейросеть для ответов на ваши вопросы. Вам нужно выбрать «поставщика»:

**Вариант A — OpenRouter** (рекомендуется, платно)
Даёт доступ к разным моделям (Claude, GPT, DeepSeek и др.) через единый API.
1. Зарегистрируйтесь на [openrouter.ai](https://openrouter.ai)
2. Пополните счёт (на 5–10$ хватит надолго)
3. Скопируйте API-ключ
4. В Zed: настройте `"assistant"` → `"provider"` → `"open_router"`, вставьте ключ

**Вариант B — Ollama** (бесплатно, работает локально)
Модели запускаются на вашем компьютере — ничего никуда не отправляется.
1. Установите [Ollama](https://ollama.com)
2. Скачайте модель: `ollama pull qwen2.5:7b` (или `llama3.1:8b`)
3. В Zed: настройте `"assistant"` → `"provider"` → `"lm_server"`, укажите `http://localhost:11434`

**Вариант C — Mistral** (платно, через их API)
1. Зарегистрируйтесь на [console.mistral.ai](https://console.mistral.ai)
2. Получите API-ключ
3. В Zed: настройте `"assistant"` → `"provider"` → `"mistral"`, вставьте ключ

> **Что выбрать новичку?** Если хотите просто попробовать — Ollama (бесплатно, но может быть медленно на слабых ПК). Если хотите максимальное качество — OpenRouter с моделью DeepSeek или Claude.

### 3. Открыть проект в Zed

Откройте папку `CoursePlanMaker` в Zed. Проект уже содержит всё, что нужно агенту для работы: шаблоны, базу данных, скрипты и инструкции (навыки).

### 4. Начать диалог

Откройте чат с ассистентом (Ctrl+Enter / Cmd+Enter) и напишите, например:

> Создай РП для дисциплины «Теория вычислительных процессов». Год набора 2026, профиль «Программное обеспечение вычислительной техники и автоматизированных систем»

Агент задаст уточняющие вопросы — отвечайте на них. Через несколько минут готовый файл появится в папке `docs/`.

## Что пока не работает (и почему)

### База данных не заполнена

В папке `исходные_данные/учебные_планы/` лежат PDF с учебными планами по разным направлениям, годам и формам обучения. Из них нужно извлечь дисциплины с часами, семестрами и ЗЕТ, чтобы заполнить `db/disciplines.db`.

**Проблема:** учебные планы — это PDF, сгенерированные из Excel. Таблицы в них — по сути картинки с текстом. Команда `pdftotext` выдаёт сырой текст, где колонки и цифры перемешаны, а LLM (нейросеть) путается, пытаясь разобрать, где часы лекций, где лабораторных, а где СРС.

**Как это решается сейчас:** человек открывает PDF, смотрит на таблицу глазами и диктует агенту: «дисциплина Х, 144 часа, 32 лк + 32 лр + 80 срс, 6 семестр». Агент записывает это в БД.

**Если бы учебные планы были в Excel (XLSX):** проблема бы решилась — Excel-таблицы парсятся программно без участия нейросети, структура колонок читается чётко. Это следующий шаг развития проекта.

### Создание ФОС по PDF с РП

Если у вас есть готовая РП в PDF, агент не сможет на её основе создать ФОС. Но если РП в Markdown (наш формат) — всё работает.

### Что уже есть в исходных данных

Папка `исходные_данные/` содержит:

- **Шаблоны и примеры РП/ФОС** в DOCX и PDF — их можно использовать как референс при создании новых документов
- **Мониторинг РПД** (XLSX) — сводка по всем дисциплинам факультета: какие РП/ФОС есть, какие отсутствуют, замечания
- **Перечень образовательных программ** (HTML) — выгрузка с сайта ЗабГУ с таблицами компетенций по годам
- **Учебные планы** (PDF) — те самые, которые пока не удалось распарсить

## Часто задаваемые вопросы

**Агент что-то сломает в моих файлах?**
Нет. Агент только создаёт новые файлы в `docs/`. Он ничего не удаляет и не перезаписывает без вашего явного согласия.

**Нужно ли знать программирование?**
Нет. Всё общение — на русском языке, обычными словами. Проект делался для преподавателей, а не для разработчиков.

**Агент ошибся в часах — что делать?**
Скажите: «в таблице 3.1.1 в строке Раздел 1 должно быть 16 часов вместо 12». Агент исправит и перепроверит.

**Можно ли использовать свой шаблон?**
Да, шаблон в `Шаблоны/рабочая_программа_дисциплины.md`. Можете отредактировать его — агент будет использовать вашу версию.

**Сколько стоит использование?**
Проект бесплатный. Платным может быть только доступ к нейросети (OpenRouter, Mistral). Ollama — полностью бесплатно.
